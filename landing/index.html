📖 Sobre o projeto

Esse projeto nasceu de uma necessidade real: eu sempre esqueço quais filmes já assisti e as notas que dei pra eles. Então resolvi criar essa aplicação fullstack simples mas funcional.

O backend foi feito com Flask, usando blueprints pra organizar as rotas (porque ninguém merece um app.py de 500 linhas). A API tem documentação Swagger automática e validação de dados com Pydantic.

O frontend é Vanilla JavaScript mesmo - sem framework, porque as vezes menos é mais. A interface consome a API e mostra feedback em tempo real.

⚠️ Nota do dev: Esse código tem algumas "gambiarras" propositais (tipo o banco de dados em memória), mas foi feito pra demonstrar conceitos reais de integração fullstack.

🚀 Como executar o projeto

🐍

Backend (Flask)

1. Clone o repositório

git clone [seu-repo]

2. Entre na pasta do backend

cd filmes-app/backend

3. Instale as dependências

pip install -r requirements.txt

4. Rode o servidor

python app.py

✅ A API vai rodar em http://localhost:5000

📚 Swagger em http://localhost:5000/swagger

🎨

Frontend (HTML/JS)

1. Entre na pasta do frontend

cd filmes-app/frontend

2. Só abrir o arquivo

index.html

Ou use um servidor simples:

python -m http.server 8000

🌐 Acesse http://localhost:8000 (ou direto pelo arquivo)

3. Aplicação Desktop (Flet)

Entre na pasta do app Flet e instale as dependências:

cd filmes-app/flet_apppip install -r requirements.txt

Execute a aplicação:

python main.py

📱 O app Flet será aberto em uma janela nativa (Windows/macOS/Linux)

⚡ Interface moderna com Material Design e suporte a temas

🔄 Funcionalidades: Listagem, adição e exclusão de filmes

📄 Landing page: você já está nela!
Basta abrir o arquivo landing/index.html no navegador.

🔌 Endpoints disponíveis

GET /api/filmes/

Lista todos os filmes cadastrados (ordenados por nota)

GET /api/filmes/{id}

Busca um filme específico pelo ID

POST /api/filmes/

Cadastra um novo filme (com validação: título não pode ser genérico, ano entre 1888 e 2026)
